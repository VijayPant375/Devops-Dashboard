Key Docker Concepts

🖼️ Docker Image

A read-only, layered blueprint of your application — OS, runtime, dependencies, and code baked into one portable artifact.

📦 Container

A running instance of an image. Isolated, lightweight, and ephemeral. Starts in milliseconds and runs identically anywhere.

📄 Dockerfile

A declarative script of instructions that tells Docker how to build your image layer by layer, from base OS to final entrypoint.

🐳 DockerHub

A cloud registry for Docker images. Push your image here and pull it on any server in the world — including your EC2 instance.
